Tarea 1 · La barra de navegación

El estudio necesita una barra profesional que funcione en celular
+20 XP
  1. Crea el archivo mision1.html con la estructura HTML base. Vincula Bootstrap 5 desde CDN en el <head> y el bundle JS al final del <body>.
  2. Construye una barra de navegación usando el componente navbar. El nombre de tu startup ficticia va como navbar-brand.
  3. La barra debe ser responsive: en pantallas grandes se muestra completa; en pantallas pequeñas, colapsa. Investiga navbar-expand-lg y navbar-toggler. ¿Qué atributos conectan el botón hamburguesa con el menú colapsable?
  4. Agrega al menos 4 links (nav-item + nav-link). Uno con clase active y otro con disabled.
  5. Uno de los links debe ser un dropdown con al menos 3 sub-opciones. Investiga dropdown-toggle, dropdown-menu y dropdown-item.
  6. Elige el esquema de colores. Recuerda que el color del texto y el del fondo son clases independientes.

Tarea 2 · Anuncio de lanzamiento

"El juego se lanza en 30 días. ¡El mundo debe saberlo!"
+10 XP
  1. Agrega debajo de la navbar un alert con la fecha de lanzamiento del juego.
  2. Usa una variante de color que transmita urgencia o emoción. Prueba alert-warning, alert-danger, alert-info y elige.
  3. El alert debe ser descartable: el usuario puede cerrarlo. Investiga data-bs-dismiss="alert" y btn-close. ¿Qué clase extra necesita el div del alert para que el botón funcione?

Tarea 3 · Galería de screenshots

"Los fans quieren ver el juego antes de comprarlo"
+15 XP
  1. Crea un carousel con al menos 3 slides. Como no tienes imágenes reales, usa bloques SVG inline con colores distintos y emojis, igual que en los ejemplos de clase.
  2. Debe incluir: indicadores (puntitos de progreso), controles anterior/siguiente, y al menos un caption con el nombre de la escena del juego.
  3. El carrusel debe reproducirse automáticamente. Investiga el atributo data-bs-ride="carousel".

Tarea 4 · Características del juego

"¿Por qué este juego es único? ¡Muéstralo en cards!"
+15 XP
  1. Crea al menos 3 cards con características del juego (ej: Multijugador, 50 niveles, Banda sonora épica).
  2. Usa row + col- para grilla responsive: 1 por fila en móvil, 2 en tablet, 3 en desktop.
  3. Cada card debe tener card-header, card-body con título y descripción, y un botón en el card-footer.
  4. Usa h-100 en las cards. ¿Qué clase del col- padre también es necesaria para igualar alturas?
  5. Personaliza el borde de al menos una card con las utilidades border-*.

Tarea 5 · Leaderboard global

"Los 5 mejores jugadores del mundo. Todos quieren verlo."
+15 XP
  1. Crea una tabla con los 5 mejores jugadores ficticios. Columnas: Posición 🏅, Nombre, Puntuación y País.
  2. Aplica al menos 3 de estos modificadores: table-striped, table-hover, table-bordered, table-dark.
  3. Agrega un caption visible a la tabla.
  4. Envuelve la tabla en table-responsive para que no rompa en móvil.

Tarea 6 · Formulario de pre-registro

"El juego no salió, pero la gente ya puede anotarse a la beta"
+20 XP
  1. Crea un formulario dentro de una card con shadow.
  2. Incluye: form-control para nombre y email, un form-select para plataforma (PC / PS5 / Xbox / Mobile), y un form-check para aceptar los términos.
  3. Usa la grilla dentro del formulario: Nombre y Email lado a lado en desktop (revisa el ejemplo "Adopta un Gatito" de clase).
  4. Activa la validación visual: novalidate en el <form>, campos required y mensajes invalid-feedback. Investiga cómo activar was-validated con JavaScript al intentar enviar.

Tarea 7 · Modal del tráiler

"Un botón épico que revela el tráiler del juego"
+20 XP
  1. Agrega un botón grande con el texto "▶ Ver Tráiler Oficial". Debe abrir un modal al hacer clic.
  2. Ábrelo usando solo atributos HTML: data-bs-toggle="modal" + data-bs-target. Sin JavaScript propio.
  3. El modal debe tener modal-header con título y botón de cierre, modal-body con descripción o iframe de YouTube, y modal-footer con "Cerrar" y "Pre-registrarme".
  4. Usa modal-lg o modal-xl en el modal-dialog para hacerlo más grande.

Tarea 8 · Stats con tooltips

"Algunos números del juego necesitan contexto al hacer hover"
+10 XP
  1. Crea al menos 3 stats del juego visibles (ej: "4.9 ★", "2M jugadores", "50h de contenido") usando badge o btn.
  2. Cada uno debe mostrar un tooltip al pasar el mouse usando los atributos data-bs-toggle="tooltip" y title.
  3. Importante: los tooltips no se activan solos. Necesitas inicializarlos con JavaScript usando querySelectorAll y un bucle, igual que en el ejemplo de clase. Sin este paso, no aparecerán.

Tarea 5 · Carta de precios

"Todo el menú con sus precios, bien ordenado"

  3. Investiga cómo cambiar el color del modal-header con las utilidades de fondo de Bootstrap.

ENTREGA FINAL

Publicar en GitHub Pages

Tu trabajo no existe si nadie puede verlo. Así se publica gratis.

Crea un repositorio público en GitHub

Ve a github.com → botón verde "New" → nombre descriptivo (ej: bootstrap-battle) → selecciona Public → clic en "Create repository".

Sube tus archivos

Opción A — interfaz web: en la página del repo clic en "uploading an existing file" → arrastra mision1.html, mision2.html y README.md → clic en "Commit changes".

Opción B — Git en terminal:

git init git add mision1.html mision2.html README.md git commit -m "Bootstrap Battle completado" git branch -M main git remote add origin https://github.com/TU_USUARIO/TU_REPO.git git push -u origin main
Activa GitHub Pages

En tu repo ve a Settings → sección Pages (menú lateral) → en Source elige "Deploy from a branch" → Branch: main, carpeta: / (root) → clic en Save.

Espera ~2 min y accede a tu sitio

GitHub mostrará la URL publicada en la misma sección Pages:

https://TU_USUARIO.github.io/TU_REPO/mision1.html https://TU_USUARIO.github.io/TU_REPO/mision2.html

⏳ Si no carga de inmediato, espera 1–2 minutos y recarga la página.

Entrega el link

Copia la URL de tu GitHub Pages y entrégala en la plataforma del curso. El link debe abrir directamente tu trabajo publicado, no el código fuente.

¡Listo! Tienes un sitio real publicado en internet. Agrégalo a tu portafolio o LinkedIn 🚀
