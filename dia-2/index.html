MINI 01 Mi primer color
01 Cambia el color principal de Bootstrap +60 pts ▼
📂 Archivo a abrir
📄 ejercicio-01/estilos.scss
Cambia el valor de $primary a un color de tu elección. Luego abre ejercicio-01/index.html en el navegador y verifica que los botones cambien.
💡 Cómo hacerlo — sintaxis @use
scss
// Con @use, las variables van en el bloque "with ()" // No se definen antes del @use como con @import @use "../scss/bootstrap" as * with ( $primary: #e63946, // ← cambia este valor ); // Después del @use puedes usar las variables de Bootstrap: .mi-clase { color: $primary; // disponible gracias a "as *" padding: $spacer; }
🎯 Qué debe pasar
✓ Verificación
Al abrir el HTML en el navegador, todos los botones .btn-primary, los badges, los links activos y los bordes de foco deben aparecer en tu nuevo color. Sin tocar el HTML.

MINI 02 Paleta completa
02 Controla los 8 colores del sistema +80 pts ▼
📂 Archivo a abrir
📄 ejercicio-02/estilos.scss
Bootstrap tiene 8 colores semánticos. Cambia todos al mismo tiempo en el with () para crear una paleta coherente de marca.
💡 Cómo hacerlo — los 8 colores
scss
@use "../scss/bootstrap" as * with ( $primary: #6c63ff, // color principal $secondary: #2d2d2d, // color secundario $success: #06d6a0, // éxito / confirmación $info: #48cae4, // información $warning: #ffd166, // advertencia $danger: #ef476f, // error / peligro $light: #f8f9fa, // fondos claros $dark: #1a1a2e, // fondos oscuros ); // Bootstrap genera automáticamente para cada color: // .btn-primary .btn-secondary .btn-success ... // .bg-primary .bg-dark .bg-light ... // .text-primary .text-danger .alert-warning ...
🎯 Qué debe pasar
✓ Verificación
Abre ejercicio-02/index.html. Todos los componentes deben reflejar tu paleta: alertas, botones, badges, tablas con .table-success, etc. Sin tocar una sola línea de HTML.
✅ Marca tu progreso
MINI 03 Tu color custom en el sistema
03 Agregar un color propio a $theme-colors +80 pts ▼
📂 Archivo a abrir
📄 ejercicio-03/estilos.scss
Bootstrap genera sus clases (.btn-X, .bg-X, .text-X…) a partir del mapa $theme-colors. Al configurar ese mapa en el with () puedes agregar tus propios colores y Bootstrap generará todas las variantes automáticamente.
💡 Cómo hacerlo — configurar $theme-colors
scss
@use "../scss/bootstrap" as * with ( $theme-colors: ( // ── Los 8 de Bootstrap (mantenlos) ── "primary": #6c63ff, "secondary": #6c757d, "success": #198754, "info": #0dcaf0, "warning": #ffc107, "danger": #dc3545, "light": #f8f9fa, "dark": #212529, // ── Tu color custom ───────────────── "marca": #ff6584, // ← Bootstrap genera .btn-marca, .bg-marca... ), );
🎯 Qué debe pasar
✓ Verificación
En ejercicio-03/index.html busca elementos con clase .btn-marca, .bg-marca o .badge.bg-marca. Deben aparecer en tu color custom sin que hayas escrito CSS manual para ellos.
✅ Marca tu progreso

MINI 05 Espaciado y decoración
05 Bordes, sombras y espaciado extendido +60 pts ▼
📂 Archivo a abrir
📄 ejercicio-05/estilos.scss
Ajusta los bordes redondeados, las sombras de las tarjetas y agrega nuevos niveles al sistema de espaciado. Cuando termines, clases como .p-6 y .m-7 existirán en el CSS.
💡 Cómo hacerlo
scss
@use "../scss/bootstrap" as * with ( // Bordes redondeados $border-radius: 0.5rem, // 8px (default: 0.375rem) $border-radius-sm: 0.25rem, $border-radius-lg: 1rem, // 16px (default: 0.5rem) $border-radius-xl: 1.5rem, // Sombras de cajas $box-shadow-sm: #{0 1px 4px rgba(0, 0, 0, .06)}, $box-shadow: #{0 4px 16px rgba(0, 0, 0, .10)}, $box-shadow-lg: #{0 8px 32px rgba(0, 0, 0, .14)}, // Espaciado extendido (agrega niveles 6, 7, 8) $spacers: ( 0: 0, 1: .25rem, 2: .5rem, 3: 1rem, 4: 1.5rem, 5: 3rem, 6: 4rem, // ← nuevo .p-6, .m-6 7: 6rem, // ← nuevo .p-7, .m-7 8: 8rem, // ← nuevo .p-8, .m-8 ), );
🎯 Qué debe pasar
✓ Verificación
En ejercicio-05/index.html las tarjetas deben tener bordes más redondeados y sombras suaves. En el DevTools (F12 → pestaña Elements) prueba agregar la clase .p-6 a un elemento — debe aplicar 4rem de padding.
✅ Marca tu progreso
